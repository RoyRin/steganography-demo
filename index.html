How It Works

This tool hides text in an image using Least Significant Bit (LSB) steganography.


Encoding: Each character of your message is converted to its 8-bit binary representation. Each bit replaces the LSB of one color channel (R, G, or B) of successive pixels. A 32-bit length header is stored first, followed by the message bits. The change to each pixel is at most ±1 in one channel — invisible to the naked eye.


Decoding: The LSBs are read back in the same order. The length header tells us how many characters to extract, then those bits are reassembled into text.


Capacity: Each pixel stores 3 bits (one per RGB channel). An image of W × H pixels can hold floor(W × H × 3 / 8) - 4 characters (minus 4 bytes for the length header).
